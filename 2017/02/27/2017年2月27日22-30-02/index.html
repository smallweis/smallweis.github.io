微信公众号的填坑之路——公众号基础篇

发表于 2017-02-27 | 分类于 微信 | | 阅读次数

文本摘要：微信公众号开发过程中的细小坑，坑的你不要不要的，一起来踩踩吧!

用户登录

微信公众号后台官方网站 https://mp.weixin.qq.com
首先后台的登录需要管理员每次扫描二维码，多个开发人员一起开发非常不方便，你可以去授权多个账号拥有开发权限，及其的方便

基本设置


微信公众号的基本设置，它包含了俩个主要的参数AppID和AppSecret。再调用大部分接口时都会用到，AppSecret只生成一次，所以需要记录到本地以防遗忘！

服务器的设置


前俩个参数是我们所关注的，也是比较坑的地方
1.url必须备案经过的域名(个人企业均可)
2.不需要填写http://请求头
3.url填写的服务器地址必须是自己填写的Token经过微信的签名处理，把签名文本返回(签名微信官方有详细的介绍——简单易懂)

网页授权登录


这里也是比较坑的,有时候我们按照微信官方配置的文档配置了登录链接，但是会报redict_uri参数错误,这时候需我们去填写登录的授权回调地址，在接口设置里面的网页授权获取用户信息，点击修改即可进入填写页面


1.同样url的域名必须经过备案
2.下载授权文件可放到主目录或者二级目录(其他目录读取不到)
3.url的地址是授权文件的存放地址

获取用户信息的get请求乱码问题

微信服务器默认的编码是UTF-8，而大多是httpget请求都是国际通用编码，get到io流时要要设置流的编码格式，java为例

微信全局常见错误返回码

Table Are
-1 系统繁忙，此时请开发者稍候再试
0 请求成功
40003 不合法的OpenID，请开发者确认OpenID（该用户）是否已关注公众号，或是否是其他公众号的OpenID
40013 不合法的AppID，请开发者检查AppID的正确性，避免异常字符，注意大小写
40014 不合法的access_token，请开发者认真比对access_token的有效性（如是否过期），或查看是否正在为恰当的公众号调用接口
